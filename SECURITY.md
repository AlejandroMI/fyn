# Security Policy

## Supported version

Fyn is an experimental project. Only the latest commit on `main` receives security fixes.

## Reporting a vulnerability

Please do not open a public issue for suspected vulnerabilities or exposed credentials.

Report them privately to `support@fynfyn.top` with:

- the affected endpoint or component;
- reproduction steps;
- the likely impact; and
- any suggested remediation.

Do not access data that is not yours, degrade the public service, or generate abusive traffic against upstream property portals while testing.

We will acknowledge a report as soon as practical and coordinate disclosure after a fix is available.
