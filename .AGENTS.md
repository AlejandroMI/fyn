This file is the fast-context map for agents working in this repo.
Use it when context is partial or stale.

1. `docs/backlog.md` (what is in progress now)
2. `docs/fyn-foundation.md` (product + engineering principles)

4. `docs/chatgpt-connector-research-2026-02-24.md` (current ChatGPT app/MCP constraints)

```bash

```

## 3) Current Product Direction (Do Not Drift)

- Brand: **Fyn** (Find Your Nest)
- Primary distribution target: **ChatGPT app** via remote MCP endpoint
- Deployment default: **Vercel**
- MCP contract is **model-driven**:
  - structured constraints are primary
  - backend executes deterministically
  - parser/NL interpretation is legacy fallback only
- Build order:
  1. Vercel MCP endpoint
  2. ChatGPT app connection end-to-end
  3. Landing CTA -> connector URL
  4. Multi-portal expansion
- Feedback ops are handled by founder; engineering stays build-focused.

## 4) Engineering Guardrails

- Do not overengineer:
  - prefer simple modules and explicit contracts
  - avoid speculative abstractions
  - ship vertical slices that can be demoed
- Keep hard constraints deterministic and enforced before ranking.
- Preserve explainability (`why_matched`) and source deep links.
- If upstream portal behavior changes, fail clearly with stable internal error codes.
- Keep scraping practical for prototype velocity; hardening/compliance follows right after first live path.

- Behavior works through the MCP tool path, not just internal functions.
- Typecheck + tests pass.
- Backlog is updated (`docs/backlog.md`) when priorities/status change.
