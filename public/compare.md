# Fyn vs. single-portal property search

Fyn is a normalized multi-portal layer for AI agents. Use it when one request should cover several Spanish property sources with consistent inputs, source attribution, match explanations, and coverage diagnostics.

Use Idealista, Fotocasa, or another portal directly when the user needs that portal's complete first-party data, account, alerts, or exclusive features. Build a custom integration when licensed data, guarantees, storage, write operations, or transactional workflows are required.

Fyn reduces integration work but does not guarantee exhaustive coverage because public portals can change or block automated access.
